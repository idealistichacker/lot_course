

# 🚀 项目技术规格书：云脑边存 - 海滨阳台无人值守气象与资产哨兵

## 1. 项目概述与约束条件

- **核心定位**：一个利用“多模态云端大模型 + 边缘 RAG 知识库”的 AIoT 决策中枢。
- **解决痛点**：在用户高强度上课或拉练无法分心时，通过摄像头被动静默感知海滨宿舍阳台的天气突变，保护昂贵资产（如竞速跑鞋），并在临界点触发跨次元的移动端预警。
- **开发周期约束**：极限 14 天（需避开复杂的机械执行机构与繁琐的网络鉴权，聚焦于业务流拆解与大模型 API 交互）。

## 2. 物理硬件与网络拓扑

- 

  **感知节点 (终端)**：**ESP32-S3 CAM** + **DHT11 温湿度传感器**。负责底层物理世界的视觉感知与环境数据采集 。

  

  

- 

  **边缘计算中枢 (边端)**：**Raspberry Pi 4 Model B 8GB**。作为系统的边缘大脑，负责运行本地后端服务、轻量级向量数据库，以及处理大模型交互 。

  

  

- **局域网中继 (网关)**：一台常驻宿舍的**闲置旧安卓手机**。开启 24/7 个人热点，为 ESP32-S3 和树莓派提供稳定的 DHCP 与互联网接入。

- **网络锚点策略**：在树莓派与 ESP32-S3 上强制绑定**静态 IP**，从底层杜绝因 IP 漂移导致的数据冒险与失联。

## 3. 核心软件技术栈

- 

  **终端控制层**：使用基于 C/C++ 的 Arduino IDE 编写底层逻辑 。

  

  

- 

  **边缘检索层 (树莓派)**：Python 3.8+ 环境 。引入轻量级向量数据库 ChromaDB 进行私有数据的持久化存储 。使用微型文本嵌入模型（如 `all-MiniLM-L6-v2`）提取特征向量 。

  

  

- 

  **云端认知层**：调用兼容 OpenAI 接口格式的云端多模态大模型 API（如 SiliconFlow 平台托管的 Qwen-VL 等视觉模型） 。

  

  

- **消息推送层**：极简 Webhook（如 Server酱或企业微信机器人），直接向移动端下发通知。

## 4. RAG 知识库 (随身小抄) 结构定义

系统将在 ChromaDB 中切块存入以下极高密度的 Markdown 数据 ：



1. **课表与节律状态**：“周二下午 14:00-17:00 满课，手机静音”或“本周末计划跑 30km LSD 拉练”。
2. **气象与热力学规则**：植入露点温度近似公式 $T_{dp} \approx T - \frac{100 - RH}{5}$ 等硬核物理机制，判断海滨盐雾与凝露灾害的临界点。
3. **资产脆弱性指标**：PEBAX 发泡材质跑鞋对高湿、暴雨及海风的耐受度红线。

## 5. 系统核心业务流水线 (Pipeline)

要求 Agent 按照以下 4 个阶段生成代码模块：

- **Step 1: 被动感知与 HTTP 推流 (C/C++)**

  ESP32-S3 CAM 设定定时器（如每 15 分钟唤醒一次），拍摄窗外及晾衣架的 `.jpg` 图像，读取 DHT11 温湿度，通过 HTTP POST 请求将多模态数据发往树莓派的 Python 极简 API 接口。

- **Step 2: 云脑视觉解析 (Python API)**

  树莓派接收到数据后，调用云端多模态大模型（VLM）API，要求 AI 识别图片中的核心要素并返回结构化数据（例：“图像中出现大面积乌云且晾衣架上存在运动鞋”）。

- **Step 3: 边存 RAG 推理与 Prompt 拼装 (Python 脚本)** 树莓派根据 VLM 的视觉结果和 DHT11 的高湿度数据，检索本地 ChromaDB。将“当前物理环境 + 视觉识别结果 + 用户的课表状态 + 跑鞋材质学”拼装成超级 Prompt 再次请求大模型进行逻辑决策 。

  

  

- **Step 4: 跨次元物理制裁 (Webhook)**

  若大模型判定“鞋子在外面 + 马上要下雨 + 用户正在上课”，则树莓派触发 Webhook 请求，直接向用户的手机推送最高优先级的红色警告指令。
