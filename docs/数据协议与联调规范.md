# 数据协议与联调规范（v0.1）

## 1. 终端上报请求
- Method: POST
- URL: /upload
- Content-Type: application/json

### JSON 字段
- device_id: string，设备唯一标识
- fw_version: string，固件版本
- timestamp_epoch: number，无 NTP 时可传 0
- seq: number，递增序号
- boot_count: number，启动次数
- uptime_ms: number，本次上电到发送耗时
- temp_c: number，温度（摄氏度）
- rh_pct: number，相对湿度（%）
- dew_point_c: number，露点近似值
- wifi_rssi: number，当前 Wi-Fi RSSI
- sensor_ok: boolean，传感器读取是否成功

### 示例
```json
{
  "device_id": "esp32s3-cam-01",
  "fw_version": "0.1.0",
  "timestamp_epoch": 0,
  "seq": 17,
  "boot_count": 17,
  "uptime_ms": 5843,
  "temp_c": 27.2,
  "rh_pct": 83.0,
  "dew_point_c": 23.8,
  "wifi_rssi": -56,
  "sensor_ok": true
}
```

## 2. 服务端返回
### 成功
- Status: 200
```json
{
  "ok": true,
  "server_time": "2026-04-01T08:30:00.000000+00:00",
  "ack_seq": 17,
  "next_action": "continue_sampling"
}
```

### 失败（字段缺失）
- Status: 422
```json
{
  "ok": false,
  "error": "missing_fields",
  "missing": ["temp_c"],
  "server_time": "2026-04-01T08:30:01.000000+00:00"
}
```

## 3. 重试策略
- 网络失败或状态码非 2xx：重试最多 3 次
- 退避等待：2s、4s、6s

## 4. 联调判定标准
- 连续 100 次上报成功率 >= 99%
- ACK 的 ack_seq 与上报 seq 一致
- 字段完整率 100%
